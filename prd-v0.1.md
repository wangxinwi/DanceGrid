# DanceGrid 产品需求文档（PRD v0.1）

## 1. 文档信息

- 产品名称：DanceGrid（暂定）
- 产品定位：街舞老师课表记录与课时费对账工具
- 版本：v0.1
- 目标平台：iPhone
- 开发策略：本地优先，先实现离线可用能力
- 用户范围：第一版仅单用户使用，预留未来多用户扩展位
- 文档日期：2026-07-07

## 2. 产品背景

用户是街舞老师，教学场景具有以下特点：

- 工作室分散，不同工作室的地址、课时费、发薪日、停课规则不一致
- 排课以月为单位更新，且常规课存在重复排班
- 临时代课、私教、workshop 等课程需要单独记录
- 课时费发放常常跨月，月底对账成本高
- 停课、请假、部分已收、超时未收等状态需要被准确记录

本产品的目标，是把“排课记录”和“课时费发放记录”统一到一个本地 iPhone 工具中，降低遗漏、计算错误和重复录入成本。

## 3. 产品目标

### 3.1 核心目标

1. 快速记录每节课的排课与执行情况
2. 快速汇总每个工作室、每个月的应收与实收情况
3. 支持常规课的月度复制，减少重复排班操作
4. 支持多视图查看课表，满足日常查看与月底对账两类场景

### 3.2 成功标准

- 月底对账时间从约 10 分钟缩短到约 10 秒
- 每次上课后可在极短时间内完成记录
- 大幅减少手动备忘录、手工计算和遗漏发薪的情况

## 4. 版本边界

### 4.1 v0.1 要做什么

- 工作室管理
- 课程模板与课程实例管理
- 月视图、周视图、列表视图
- 当日课程详情查看
- 课程复制
- 月度复制常规课
- 对账中心
- 本地数据存储
- 内测邀请码访问控制

### 4.2 v0.1 不做什么

- 云端同步
- 自动归档
- 导入导出
- 多端同步
- 提醒推送
- 复杂统计报表
- 课程附件的在线处理

### 4.3 发布与访问控制

v0.1 先作为封闭内测版本发布，目标席位数为 100 个。

规则：

- 仅通过邀请码进入应用
- 一个邀请码默认只对应一个席位
- 席位由 Cloudflare Workers 进行校验与分配
- 不做公开注册
- 不做 PIN 锁作为主访问方案
- 不上传课程、工作室、对账等业务数据到后端
- 只把邀请码状态、席位状态和最小必要的授权信息交给轻量后端处理

邀请码规则：

- 邀请码分为单次兑换码和批量邀请码两类
- 单次兑换码适合直接发给某一位测试用户
- 批量邀请码适合小范围邀请，但后台仍按席位数控制总量
- 邀请码可设置失效时间
- 邀请码一旦被兑换，默认绑定到当前设备或当前安装实例
- 邀请码与席位状态由 Cloudflare Worker 记录和校验
- Worker 只保存最小必要的授权状态，不保存课程、工作室和对账业务数据
- 邀请码可以在后台停用、释放或重新发放

席位管理规则：

- 初始总席位数：100
- 每个席位状态包括：可用、已预留、已兑换、已停用、已回收、已过期
- 已兑换席位保留到内测期结束，除非被手动回收
- 若用户更换设备或重新安装导致本地授权丢失，可由后台手动重置并重新发放
- 若席位满了，新用户默认进入候补名单，必须等到有席位释放后再补发邀请码
- 若某位测试用户长期不活跃，可先标记为待回收，再由后台人工收回席位
- 对明显泄露或转发的邀请码，可直接停用并重新发放新码
- 席位回收后可以重新发放给新的内测用户
- 后台只管理授权与席位，不接触本地业务数据

失效与回收规则：

- 未兑换的邀请码到期后自动失效
- 已兑换的邀请码在内测结束前默认有效
- 若服务端检测到邀请码被吊销，App 下次联网时需要重新验证
- 若某席位被回收，原设备上的授权在下一次联网校验时失效
- 回收后的席位可重新发放给候补用户

后期方案：

- 内测完成后，公开版不再使用邀请码作为主要门禁
- App Store 上线后，改为买断制
- 买断后的授权由 App Store 收据或系统购买状态决定
- 定价和收款方式先保留为待定，不在 v0.1 中锁死
- 未来如需多设备或团队版，再单独设计账户体系
- 公开版不再依赖本地 PIN，也不需要单独的邀请码输入
- 若后期推出买断版，邀请系统仅保留给内测或灰度发布使用

## 5. 用户与使用场景

### 5.1 用户角色

- 第一版主用户：你本人
- 未来扩展用户：其他街舞老师
- 内测阶段用户：受邀的 100 位测试用户

### 5.2 主要使用场景

1. 上完课后，立即记录本节课是否正常进行、是否停课、是否请假
2. 月初复制上月常规排班，快速生成本月课程实例
3. 月底进入对账中心，查看各工作室应收、实收、未收
4. 临时代课、私教、workshop 出现时，手动新增一节课程并填写金额

## 6. 产品原则

- 先快后全：第一版优先覆盖高频动作
- 本地优先：所有数据仅存本机
- 少打扰：不做强提醒，不做推送
- 可追溯：历史课程、状态和金额要能回看
- 可扩展：结构上预留未来多用户和更多课程类型扩展能力

## 7. 业务定义

### 7.1 工作室

工作室是课程发生的地点和结算主体。每个工作室可配置：

- 工作室名称
- 地址
- 基础课时费
- 课时费单位
- 发薪日
- 停课补偿比例
- 排课节数
- 联系人
- 电话或微信
- 备注
- 分组标签

### 7.2 课程类型

v0.1 支持以下课程类型：

- 常规课
- 临时代课
- 工作室私教
- 学员私教
- 小班课
- workshop

说明：

- 常规课来自课程模板，可按周重复生成
- 临时代课、私教、workshop 需要手动输入课时费
- 课程类型用于统计、筛选和对账

### 7.3 状态定义

#### 开课状态

- 待开
- 已开
- 停课
- 请假

定义：

- 请假：老师本人原因无法上课
- 停课：学员人数不足或工作室原因导致课程取消

#### 收款状态

- 未收
- 已收
- 部分已收
- 超时未收

定义：

- 部分已收：按节次维度部分结算
- 超时未收：系统根据发薪日和当前日期自动标记为提醒标签

### 7.4 状态组合规则

- 开课状态与收款状态允许独立变化
- 已开但未收允许存在
- 停课但已收允许存在
- 待开但已收不建议出现，若发生需提示用户确认

## 8. 数据模型

### 8.1 用户

第一版逻辑上按单用户处理，但数据表预留 `user_id` 字段，便于未来扩展。

字段建议：

- id
- name
- password_hash（预留，当前内测不启用）
- created_at

### 8.2 工作室

字段建议：

- id
- name
- address
- base_fee
- fee_unit
- pay_day
- cancel_compensation_ratio
- weekly_session_count
- contact_name
- contact_way
- group_name
- note
- archived_at

### 8.3 课程模板

课程模板用于常规课的重复生成。

字段建议：

- id
- studio_id
- course_type = 常规课
- weekdays
- start_time
- end_time
- content_tag
- content_description
- default_departure_minutes
- default_fee
- note
- archived_at

说明：

- 一个模板可关联多个星期几
- 模板修改不回写历史课程实例

### 8.4 课程实例

课程实例是具体某一天的一节课记录。

字段建议：

- id
- user_id
- studio_id
- template_id，可为空
- course_type
- class_date
- start_time
- end_time
- content_tag
- content_description
- fee
- departure_minutes
- class_status
- payment_status
- actual_receivable_amount
- actual_received_amount
- music_note
- attachments
- note
- created_at
- updated_at

说明：

- 临时代课、私教、workshop 的课时费由用户手动输入
- 常规课默认继承模板费用，可手动改单节金额
- 图片和文件附件可作为本地资源存储

## 9. 功能需求

### 9.1 工作室管理

功能：

- 新增工作室
- 编辑工作室
- 归档工作室
- 恢复归档工作室
- 软删除工作室

规则：

- 第一版优先使用“归档停用”，不做直接物理删除作为主流程
- 若用户执行删除，需先二次确认
- 删除前提示将影响的模板和课程实例

### 9.2 课程新增与编辑

功能入口：

- 日历页右下角新增按钮
- 课程详情页编辑按钮
- 课程卡片左滑编辑

新增方式：

- 新建常规课
- 新建临时代课
- 新建工作室私教
- 新建学员私教
- 新建小班课
- 新建 workshop

规则：

- 常规课优先选择模板生成
- 临时代课、私教、workshop 必填课时费
- 支持复制已有课程生成新课程
- 支持修改单节课程，不自动影响模板

### 9.3 月视图首页

展示内容：

- 月份切换
- 月历格子
- 当日课程列表
- 快捷新增入口
- 快捷批量操作入口

日历格子规则：

- 绿色：已开课
- 灰色：待开课
- 红色：停课

显示内容：

- 每日课程数量或状态小图标
- 课程时间
- 工作室名称

### 9.4 周视图

用途：

- 快速查看本周全部课程
- 便于临时调整排课

展示内容：

- 按星期展示
- 按时间排序
- 显示工作室、课程类型、金额、状态

### 9.5 列表视图

用途：

- 适合快速翻查历史课程
- 适合按时间顺序查看连续课程

展示内容：

- 月份内全部课程
- 支持按日期分组
- 支持快速筛选课程类型和工作室

### 9.6 课程详情页

字段展示：

- 上课日期
- 上课时间
- 工作室名称
- 工作室地址
- 课程类型
- 课程内容
- 课时费
- 开课状态
- 收款状态
- 出门提前分钟数
- 音乐备注
- 附件
- 备注

操作：

- 编辑
- 复制
- 删除
- 修改状态
- 修改金额

### 9.7 月度复制常规课

入口：

- 月切换处按钮“复制上月排班”

规则：

- 仅复制常规课模板生成的课程
- 不复制临时代课
- 不复制停课记录
- 不复制收款状态
- 不复制开课状态
- 新生成课程实例默认置为“待开 + 未收”
- 保留原课程内容、时间、备注

冲突处理：

- 若目标月已存在课程，逐条提示用户处理
- 用户可选择覆盖、跳过或手动保留

### 9.8 批量操作

v0.1 支持批量操作，但仅在明确选中范围内执行。

支持范围：

- 当前月
- 当前工作室
- 当前筛选结果
- 当前选中课程

建议操作：

- 标为正常开课
- 标为停课
- 标为课时费已发

规则：

- 批量操作前必须二次确认
- 必须显示受影响课程数量
- 必须显示涉及的工作室名称
- 支持撤销上一步批量操作

### 9.9 对账中心

展示方式：

- 按工作室分组
- 默认显示当前月份
- 支持切换历史月份
- 当前页以“课表发生月”为主，收款金额另外按“打款月”统计

汇总字段：

- 总节数
- 应收合计
- 实收合计
- 当月收款合计
- 未收差额
- 停课节数
- 补偿金额
- 预计发薪日
- 本月已过天数进度

计算规则：

- 应收金额以课程实例金额汇总
- 实收金额由课程实例的收款状态和实收金额汇总
- 当月收款合计按 `payment_time` 所在月份统计
- 未收差额为应收减实收
- 停课补偿默认 0，可手动改写
- 超时未收为系统标签，不改变原始金额
- 一键已收款只更新该工作室的上一个月份课程实例
- 若部分已收需要补齐，默认优先处理该工作室上月最新日期的一条记录

明细钻取：

- 点击工作室名称展开明细
- 明细显示日期、时间、状态、金额、备注
- 支持按月回看

### 9.10 发薪日规则

规则：

- 发薪日支持 1 到 31
- 当工作室设置为 29、30、31 号，且目标月份不存在该日期时，系统必须弹出异常提醒
- 用户可选择强制沿用并自动映射为当月最后一天，或重新设置

### 9.11 内测邀请码

v0.1 内测阶段不把本地 PIN 作为主访问方案。

目的：

- 防止未受邀用户进入应用

## 10. 非功能需求

### 10.1 平台

- iPhone 优先
- 适配常见 iPhone 尺寸

### 10.2 性能

- 月视图切换响应时间应小于 300ms
- 常见列表滚动应保持流畅

### 10.3 数据存储

- 所有数据仅存本地
- 使用本地数据库
- 不申请云端同步权限

### 10.4 安全

- 支持邀请码校验与席位控制
- 支持系统层面的 Face ID / Touch ID（若可用）

### 10.5 视觉与交互

- 整体风格偏工具型、日历型
- 以蓝、黑、白为主
- 信息展示清晰优先
- 不使用渐变
- 不使用复杂动效
- 重要信息直接可见

## 12. 关键业务流程

### 12.1 日常上课记录

1. 打开 App
2. 查看当天或当周课程
3. 进入课程详情
4. 更新开课状态、收款状态、音乐备注

### 12.2 月初复制排班

1. 进入月视图
2. 点击“复制上月排班”
3. 系统生成本月常规课
4. 若冲突，逐条处理

### 12.3 临时代课新增

1. 点击新增
2. 选择课程类型
3. 选择工作室与日期时间
4. 手动输入课时费
5. 保存

### 12.4 月底对账

1. 进入对账中心
2. 默认查看当月对账，左右切换历史月份
3. 查看每个工作室汇总
4. 核对应收、实收、未收
5. 查看当月收到的工资
6. 查看超时未收标签
7. 必要时回到课程详情修正记录

## 13. 边界与异常

- 同一天多个课程时，列表必须按时间顺序排列
- 修改模板不回写历史实例
- 修改工作室基础课时费不自动重算历史已结算课程
- 恢复旧记录时，新旧冲突必须手动处理
- 删除工作室优先改为归档停用
- 停课与请假必须保留不同语义

## 14. 验收标准

### 14.1 功能验收

- 能记录常规课与临时代课
- 能按月复制常规排班
- 能查看月、周、列表三种视图
- 能按工作室查看对账数据
- 能记录停课、请假、部分已收、超时未收

### 14.2 效率验收

- 月底对账时间明显减少
- 排课复制效率明显高于手工录入
- 上课后记录流程足够快

### 14.3 准确性验收

- 课时费计算无误
- 月度汇总无误
- 状态组合显示清晰，不混淆停课与请假

## 15. 说明与待确认项

以下内容在 v0.1 中采用了当前最稳妥的实现假设：

- 第一版按单用户设计，未来再扩展多用户
- 归档与导入导出不进入 v0.1
- 发薪日异常采用弹窗确认，不做静默修正
- 图片与文件附件作为后续增强能力，可根据实现成本调整优先级
