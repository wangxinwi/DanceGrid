# Product

## Register
product

## Users
Primary user is a street dance teacher managing studio schedules and class fees on an iPhone, usually right after class or while checking monthly settlement.

## Product Purpose
DanceGrid helps a single user record class sessions, repeat schedules, fee status, and monthly reconciliation locally on device. Success means the user can see today’s classes quickly, log a session in seconds, and reconcile a month without switching to notes, spreadsheets, or memory.

## Brand Personality
Calm, precise, efficient.
The product should feel like a reliable work tool that disappears into the task: clear, direct, and low-noise.

## Anti-references
- Decorative consumer calendar apps that favor delight over speed.
- SaaS dashboards with noisy gradients, oversized illustrations, or marketing-style metric cards.
- Overly playful scheduling apps that make routine edits feel like a game.
- Interfaces that hide core actions behind custom metaphors or unconventional controls.

## Design Principles
- Make the schedule legible first, beautiful second.
- Optimize for rapid review and rapid edit on mobile.
- Keep core data local, stable, and easy to audit.
- Use consistent language for class status, payment status, and reconciliation.
- Preserve history without forcing the user into a database mindset.

## Accessibility & Inclusion
Target WCAG AA contrast and strong touch targets for iPhone use.
Support reduced motion, clear state colors paired with text labels, and simple copy that works for quick scanning.
