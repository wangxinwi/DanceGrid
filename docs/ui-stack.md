# DanceGrid UI Stack

This project should prefer a single UI stack so AI-assisted edits stay consistent.

## Approved Stack

- React
- shadcn/ui
- Radix UI
- Lucide

## Guiding Rules

- Reuse existing UI components before creating new ones.
- Prefer shadcn/ui components for application UI.
- Use Radix UI primitives for accessibility and interaction behavior.
- Use Lucide for icons only.
- Avoid introducing another visual system unless explicitly approved.

## Suggested Component Whitelist

- `Button`
- `Input`
- `Textarea`
- `Select`
- `Checkbox`
- `RadioGroup`
- `Switch`
- `Dialog`
- `AlertDialog`
- `Popover`
- `DropdownMenu`
- `Command`
- `Tabs`
- `Sheet`
- `Tooltip`
- `Toast`
- `Badge`
- `Card`
- `Separator`
- `Table`
- `Accordion`
- `Calendar`
- `Pagination`
- `ScrollArea`
- `Slider`
- `Skeleton`

## Icon Rules

- Prefer Lucide icons.
- Use a descriptive icon name when possible.
- Keep icon usage consistent across screens.
- Avoid mixing multiple icon packs in the same product area.
